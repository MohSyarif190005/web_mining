Bagging Classifier (Ensemble Learning), Grid Search dan Random Forest#

Pengertian Bagging Classifier#

Pengklasifikasi Bagging adalah meta-estimator ensemble yang cocok dengan pengklasifikasi dasar masing-masing pada subset acak dari dataset asli dan kemudian menggabungkan prediksi individu mereka (baik dengan pemungutan suara atau rata-rata) untuk membentuk prediksi akhir. Meta-estimator seperti itu biasanya dapat digunakan sebagai cara untuk mengurangi varians dari estimator kotak hitam (misalnya, pohon keputusan), dengan memasukkan pengacakan ke dalam prosedur konstruksinya dan kemudian membuat ansambel darinya. Setiap pengklasifikasi dasar dilatih secara paralel dengan set pelatihan yang dihasilkan dengan menggambar secara acak, dengan penggantian, N contoh (atau data) dari set data pelatihan asli – di mana N adalah ukuran set pelatihan asli. Pelatihan set untuk masing-masing pengklasifikasi dasar adalah independen satu sama lain. Banyak dari data asli dapat diulang dalam set pelatihan yang dihasilkan sementara yang lain mungkin ditinggalkan.

Bagging mengurangi overfitting (varians) dengan rata-rata atau voting, namun, hal ini menyebabkan peningkatan bias, yang dikompensasi oleh pengurangan varians sekalipun.

Bagaimana Bagging bekerja pada dataset pelatihan? Cara kerja bagging pada dataset pelatihan imajiner ditunjukkan di bawah ini. Karena Bagging membuat sampel ulang set data pelatihan asli dengan penggantian, beberapa instance (atau data) mungkin ada beberapa kali sementara yang lain tidak disertakan.

Algoritma Bagging Classifier#

Algoritma ini mencakup beberapa karya dari literatur. Ketika subset acak dari dataset diambil sebagai subset acak dari sampel, maka algoritma ini dikenal sebagai Paste. Jika sampel diambil dengan penggantian, maka metode tersebut dikenal sebagai Bagging. Ketika subset acak dari dataset digambar sebagai subset acak dari fitur, maka metode ini dikenal sebagai Random Subspaces. Akhirnya, ketika penduga dasar dibangun di atas himpunan bagian dari sampel dan fitur, maka metode ini dikenal sebagai Patch Acak.

Pengertian Grid Search#

Grid Seacrh, Metode ini paling mudah untuk diterapkan dan dipahami, tetapi sayangnya tidak efisien ketika jumlah parameternya besar dan tidak terlalu dibatasi di bawah H0. Biarkan Ω* menjadi ruang parameter gangguan = (ν1, 2, …νm) di mana kita memaksimalkan nilai-p. Cara sederhana untuk mengatur pencarian grid terdiri dari mendefinisikan vektor batas bawah a = (a1, a2, …, am) dan vektor batas atas b = (b1, b2, …, bm) untuk setiap komponen . Pencarian grid melibatkan pengambilan n titik dengan jarak yang sama di setiap interval dari bentuk [ai, bi] termasuk ai dan bi. Ini menciptakan total nm titik kisi yang mungkin untuk diperiksa. Akhirnya, setelah setiap pasangan poin dihitung, maksimum dari nilai-nilai ini dipilih.

Masalah dengan jenis metode ini adalah bahwa jumlah evaluasi meningkat secara eksponensial dengan meningkatnya n dan m. Karena kita tidak dapat benar-benar mengurangi m, penurunan n adalah satu-satunya cara yang mungkin untuk memastikan bahwa metode berhenti dalam waktu yang wajar, tetapi ini mengurangi validitas solusi.

Pengertian Crawling#

Crawling adalah semacam pengambilan data dari media sosial kemudian di kumpulkan menjadi satu untuk di evakuasi dan di bentuk agar menjadi sebuah penelitian. Prosesnya cukup mudah tergantung kita ingin mengambil data dari sosial media mana. Sebagai contoh jika ingin melakukan crawling data twitter, ada 2 cara, yaitu menggunakan API atau tanpa menggunakan API. Jika menggunakan API, maka menggunakan Twint.

Melakukan Crawling Data Twitter Menggunakan Twint#

Twint adalah sebuah tools yang digunakan untuk melakukan scrapping dari aplikasi twitter yang disetting secara khusus menggunakan bahasa pemrograman Python. Twint dapat kita gunakan dan jalankan tanpa harus menggunakan API dari Twitter itu sendiri, dengan kapasitas scrapping data maksimalnya adalah 3200 tweet.

Preprocessing Text#

Data preprocessing adalah proses yang mengubah data mentah ke dalam bentuk yang lebih mudah dipahami. Proses ini penting dilakukan karena data mentah sering kali tidak memiliki format yang teratur.

Manfaat Preprocessing#

  • Membuat data lebih mudah untuk dibaca
  • Mengurangi beban representasi dalam data
  • Mengurangi durasi data mining secara signifikan
  • Mempermudah proses analisis data dalam machine learning

    • Tahapan Preprocessing#

    1. Data Cleaning
      Dalam tahap kerja ini, data dibersihkan melalui beberapa proses seperti mengisi nilai yang hilang, menghaluskan noisy data, dan menyelesaikan inkonsistensi yang ditemukan. Data juga bisa dibersihkan dengan dibagi menjadi segmen-segmen yang memiliki ukuran serupa lalu dihaluskan (binning).

    2. Data Integration
      Di sini, data dengan representasi yang berbeda disatukan dan semua konflik dalam di dalamnya diselesaikan. Tahap kerja satu ini merupakan proses lanjutan dari data cleansing dengan tujuan untuk membuat data lebih halus.

    3. Data Transformation
      Pada tahap ini, data akan dinormalisasi dan digeneralisasikan. Normalisasi sendiri adalah sebuah proses di mana perusahaan memastikan bahwa tidak ada data yang berlebihan. Semua data akan disimpan dalam satu tempat dan semua dependensinya haruslah logis. Langkah ini juga diambil untuk mentransformasikan data ke dalam bentuk yang sesuai untuk proses mining.

    4. Data Reduction
      Pada tahap ini, bisa meningkatkan efisiensi penyimpanan dan mengurangi representasi data dalam data sehingga data yang dikelola kecil dan mengurangi waktu komputasi.

    Cleaning Text#

    1. Fungsi cleaningText, menghapus mention, menghapus hashtag, menghapus RT, menghapus link, dan menghapus nomor, mengganti baris baru dengan spasi, menghapus semua tanda baca, dan menghapus berbagai macam karakter atau emoji teks dari kiri ke kanan. 

    def cleaningText(text):
    text = re.sub(r'@[A-Za-z0-9]+', '', text) # remove mentions
    text = re.sub(r'#[A-Za-z0-9]+', '', text) # remove hashtag
    text = re.sub(r'RT[\s]', '', text) # remove RT
    text = re.sub(r"http\S+", '', text) # remove link
    text = re.sub(r'[0-9]+', '', text) # remove numbers

    text = text.replace('\n', ' ') # replace new line into space
    text = text.translate(str.maketrans('', '', string.punctuation)) # remove all punctuations
    text = text.strip(' ') # remove characters space from both left and right text
    return text

    Case Folding Text#

    1. Fungsi caseFoldingText, mengubah semua huruf menjadi huruf kecil (lowercase) semua. Sementara itu, karakter lain yang bukan termasuk huruf dan angka, seperti tanda baca dan spasi dianggap sebagai delimiter. Delimiter ini bisa juga dihapus atau diabaikan dengan menggunakan perintah yang ada di Python.

    def casefoldingText(text): # Converting all the characters in a text into lower case
    text = text.lower() 
    return text

    Tokenizing Text#

    1. Fungsi tokenizingText, pemisahan string pada teks menjadi beberapa daftar token sehingga dapat digunakan untuk menganalisa teks 

    def tokenizingText(text): # Tokenizing or splitting a string, text into a list of tokens
    text = word_tokenize(text) 
    return text

    Filtering Text#

    1. Fungsi filteringText, tahap mengambil kata-kata penting dari hasil token dengan menggunakan algoritma stoplist (membuang kata kurang penting) atau wordlist (menyimpan kata penting).

    def filteringText(text): # Remove stopwors in a text
    listStopwords = set(stopwords.words('indonesian'))
    filtered = []
    for txt in text:
        if txt not in listStopwords:
            filtered.append(txt)
    text = filtered 
    return text

    Stemming Text#

    1. Fungsi stemmingText, proses menghilangkan infleksi kata ke bentuk dasarnya, namun bentuk dasar tersebut tidak berarti sama dengan akar kata (root word).

    def stemmingText(text): # Reducing a word to its word stem that affixes to suffixes and prefixes or to the roots of words
    factory = StemmerFactory()
    stemmer = factory.create_stemmer()
    text = [stemmer.stem(word) for word in text]
    return text

    def toSentence(list_words): # Convert list of words into sentence

    Random Forest dengan Decision Tree#

    Decision tree merupakan sebuah metode dari machine learning dan termasuk kedalam supervised learning atau pembelajaran terawasi. Decision tree Itu membuat prediksi, seperti bagaimana, pikiran manusia akan membuat, dalam kehidupan nyata. Ini dapat dianggap sebagai serangkaian pernyataan if-then-else dan terus membuat keputusan atau prediksi di setiap titik, seiring pertumbuhannya.

    Decision tree terlihat seperti diagram alur atau pohon terbalik. Tumbuh dari akar ke daun tetapi dengan cara terbalik. Kita dapat dengan mudah menginterpretasikan proses pengambilan keputusan / prediksi dari pohon keputusan, karena strukturnya. Berikut gambarannya:



    Decision tree dapat berupa Pohon Klasifikasi atau Pohon Regresi, berdasarkan jenis variabel target. Kelas dalam kasus pohon klasifikasi didasarkan pada prediksi mayoritas di simpul daun. Dalam kasus regresi, nilai prediksi akhir didasarkan pada nilai rata-rata di simpul daun.

    Ada berbagai algoritma Pohon Keputusan tergantung pada apakah itu split biner atau split multiway. Pemisahan Biner berarti simpul induk terbagi menjadi 2 simpul anak dan dalam pemisahan multiway, simpul induk terbagi menjadi lebih dari 2 simpul anak. Salah satu contoh dalam Decision tree yang berupa klasifikasi adalah CART (Classification and Regression Tree).

    CART atau Classification and Regression Tree berurusan dengan pohon split biner. Decision tree membagi data menjadi berbagai himpunan bagian dan kemudian membuat pemisahan berdasarkan atribut yang dipilih. Atribut ini dipilih berdasarkan kriteria homogenitas yang disebut Gini Index.

    Gini index pada dasarnya mengukur kemurnian (atau ketidakmurnian juga, kita dapat mengatakan) dari node setelah perpecahan terjadi. Artinya, ini adalah ukuran seberapa murni simpul anak dari simpul induk, setelah pemisahan.


    Pohon harus membuat pemisahan berdasarkan atribut yang mengarah ke homogenitas maksimum dari simpul anak. Rumus Gini Index adalah sebagai berikut :



    Dimana pi adalah probabilitas menemukan titik dengan label i, dan n adalah jumlah kelas/nilai resultan dari variabel target.

    Dengan demikian pohon menghitung indeks Gini pada simpul induk. Kemudian indeks Gini tertimbang pada node anak dihitung untuk setiap atribut. Sebuah atribut dipilih sedemikian rupa sehingga kita mendapatkan homogenitas maksimum di node anak dan pemisahan dilakukan. Proses ini diulang sampai semua atribut habis. Pada akhirnya kita mendapatkan apa yang kita sebut sebagai pohon Keputusan yang tumbuh sepenuhnya. Dengan demikian atribut yang mengarah pada pemisahan awal dapat dianggap sebagai atribut yang paling penting untuk prediksi.
