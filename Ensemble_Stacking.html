Data Web Twitter#

Pengertian Crawling#

Crawling adalah semacam pengambilan data dari media sosial kemudian di kumpulkan menjadi satu untuk di evakuasi dan di bentuk agar menjadi sebuah penelitian. Prosesnya cukup mudah tergantung kita ingin mengambil data dari sosial media mana. Sebagai contoh jika ingin melakukan crawling data twitter, ada 2 cara, yaitu menggunakan API atau tanpa menggunakan API. Jika menggunakan API, maka menggunakan Twint.

Melakukan Crawling Data Twitter Menggunakan Twint#

Twint adalah sebuah tools yang digunakan untuk melakukan scrapping dari aplikasi twitter yang disetting secara khusus menggunakan bahasa pemrograman Python. Twint dapat kita gunakan dan jalankan tanpa harus menggunakan API dari Twitter itu sendiri, dengan kapasitas scrapping data maksimalnya adalah 3200 tweet.

Preprocessing Text#

Data preprocessing adalah proses yang mengubah data mentah ke dalam bentuk yang lebih mudah dipahami. Proses ini penting dilakukan karena data mentah sering kali tidak memiliki format yang teratur.

Manfaat Preprocessing#

  • Membuat data lebih mudah untuk dibaca
  • Mengurangi beban representasi dalam data
  • Mengurangi durasi data mining secara signifikan
  • Mempermudah proses analisis data dalam machine learning

    • Tahapan Preprocessing#

    1. Data Cleaning
      Dalam tahap kerja ini, data dibersihkan melalui beberapa proses seperti mengisi nilai yang hilang, menghaluskan noisy data, dan menyelesaikan inkonsistensi yang ditemukan. Data juga bisa dibersihkan dengan dibagi menjadi segmen-segmen yang memiliki ukuran serupa lalu dihaluskan (binning).

    2. Data Integration
      Di sini, data dengan representasi yang berbeda disatukan dan semua konflik dalam di dalamnya diselesaikan. Tahap kerja satu ini merupakan proses lanjutan dari data cleansing dengan tujuan untuk membuat data lebih halus.

    3. Data Transformation
      Pada tahap ini, data akan dinormalisasi dan digeneralisasikan. Normalisasi sendiri adalah sebuah proses di mana perusahaan memastikan bahwa tidak ada data yang berlebihan. Semua data akan disimpan dalam satu tempat dan semua dependensinya haruslah logis. Langkah ini juga diambil untuk mentransformasikan data ke dalam bentuk yang sesuai untuk proses mining.

    4. Data Reduction
      Pada tahap ini, bisa meningkatkan efisiensi penyimpanan dan mengurangi representasi data dalam data sehingga data yang dikelola kecil dan mengurangi waktu komputasi.

    Cleaning Text#

    1. Fungsi cleaningText, menghapus mention, menghapus hashtag, menghapus RT, menghapus link, dan menghapus nomor, mengganti baris baru dengan spasi, menghapus semua tanda baca, dan menghapus berbagai macam karakter atau emoji teks dari kiri ke kanan. 

    def cleaningText(text):
    text = re.sub(r'@[A-Za-z0-9]+', '', text) # remove mentions
    text = re.sub(r'#[A-Za-z0-9]+', '', text) # remove hashtag
    text = re.sub(r'RT[\s]', '', text) # remove RT
    text = re.sub(r"http\S+", '', text) # remove link
    text = re.sub(r'[0-9]+', '', text) # remove numbers

    text = text.replace('\n', ' ') # replace new line into space
    text = text.translate(str.maketrans('', '', string.punctuation)) # remove all punctuations
    text = text.strip(' ') # remove characters space from both left and right text
    return text

    Case Folding Text#

    1. Fungsi caseFoldingText, mengubah semua huruf menjadi huruf kecil (lowercase) semua. Sementara itu, karakter lain yang bukan termasuk huruf dan angka, seperti tanda baca dan spasi dianggap sebagai delimiter. Delimiter ini bisa juga dihapus atau diabaikan dengan menggunakan perintah yang ada di Python.

    def casefoldingText(text): # Converting all the characters in a text into lower case
    text = text.lower() 
    return text

    Tokenizing Text#

    1. Fungsi tokenizingText, pemisahan string pada teks menjadi beberapa daftar token sehingga dapat digunakan untuk menganalisa teks 

    def tokenizingText(text): # Tokenizing or splitting a string, text into a list of tokens
    text = word_tokenize(text) 
    return text

    Filtering Text#

    1. Fungsi filteringText, tahap mengambil kata-kata penting dari hasil token dengan menggunakan algoritma stoplist (membuang kata kurang penting) atau wordlist (menyimpan kata penting).

    def filteringText(text): # Remove stopwors in a text
    listStopwords = set(stopwords.words('indonesian'))
    filtered = []
    for txt in text:
        if txt not in listStopwords:
            filtered.append(txt)
    text = filtered 
    return text

    Stemming Text#

    1. Fungsi stemmingText, proses menghilangkan infleksi kata ke bentuk dasarnya, namun bentuk dasar tersebut tidak berarti sama dengan akar kata (root word).

    def stemmingText(text): # Reducing a word to its word stem that affixes to suffixes and prefixes or to the roots of words
    factory = StemmerFactory()
    stemmer = factory.create_stemmer()
    text = [stemmer.stem(word) for word in text]
    return text

    to Sentence#

    1. Fungsi toSentence, Ubah daftar kata menjadi kalimat

    def toSentence(list_words): # Convert list of words into sentence
    sentence = ' '.join(word for word in list_words)
    return sentence

    Klasifikasi#

    Klasifikasi merupakan proses untuk menemukan model atau fungsi yang menjelaskan atau membedakan konsep atau kelas data, dengan tujuan untuk dapat memperkirakan kelas dari suatu objek yang labelnya tidak diketahui. Model itu sendiri bisa berupa aturan “jika-maka”, berupa decision tree, formula matematis atau neural network. Metode-metode klasifikasi antara lain C4.5, RainForest, Naïve Bayesian, neural network, genetic algorithm, fuzzy, case-based reasoning, dan k-Nearest Neighbor.

    Metode K-NN#

    Algoritma KNN merupakan algoritma klasifikasi yang bekerja dengan mengambil sejumlah K data terdekat (tetangganya) sebagai acuan untuk menentukan kelas dari data baru. Algoritma ini mengklasifikasikan data berdasarkan similarity atau kemiripan atau kedekatannya terhadap data lainnya.
    Secara umum, cara kerja algoritma KNN adalah sebagai berikut.

    1. Tentukan jumlah tetangga (K) yang akan digunakan untuk pertimbangan penentuan kelas.

    2. Hitung jarak dari data baru ke masing-masing data point di dataset.

    3. Ambil sejumlah K data dengan jarak terdekat, kemudian tentukan kelas dari data baru tersebut.

    Klasifikasi Random Forest#

    Random Forest adalah model ensemble kuat yang dibangun dengan sejumlah besar Pohon Keputusan. Ini mengatasi kekurangan dari pohon keputusan tunggal di samping beberapa keuntungan lainnya.     Ensemble Stacking#

    Pengertian Ensemble Stacking#

    Ensemble Stacking adalah metode yang menggunakan beberapa algoritma atau model pembelajaran untuk menghasilkan satu model prediksi yang optimal. Model yang dihasilkan memiliki kinerja yang lebih baik dibandingkan dengan basis peserta didik yang diambil sendiri. Aplikasi lain dari pembelajaran ensemble juga termasuk memilih fitur penting, fusi data, dll. Teknik ensemble dapat diklasifikasikan menjadi Bagging, Boosting, and Stacking.

    Algoritma Ensemble Stacking#

    Bagging dan boosting menggunakan pelajar lemah yang homogen untuk ensamble, Stacking sering menganggap pelajar lemah yang heterogen, mempelajarinya secara paralel, dan menggabungkannya dengan melatih meta-pelajar untuk menghasilkan prediksi berdasarkan prediksi pelajar lemah yang berbeda. Pelajar meta memasukkan prediksi sebagai fitur dan target menjadi nilai kebenaran dasar dalam data D (Gbr 2.), ia mencoba mempelajari cara terbaik untuk menggabungkan prediksi input untuk membuat prediksi output yang lebih baik.

    Dalam ansambel rata-rata misalnya. Random Forest, model menggabungkan prediksi dari beberapa model terlatih. Keterbatasan pendekatan ini adalah bahwa setiap model memberikan kontribusi jumlah yang sama untuk prediksi ensemble, terlepas dari seberapa baik model dilakukan. Pendekatan alternatif adalah ensemble rata-rata tertimbang, yang menimbang kontribusi setiap anggota ensemble dengan kepercayaan atas kontribusi mereka dalam memberikan prediksi terbaik. Ansambel rata-rata tertimbang memberikan peningkatan atas ansambel rata-rata model.

    Generalisasi lebih lanjut dari pendekatan ini adalah mengganti jumlah tertimbang linier dengan Regresi Linier (masalah regresi) atau Regresi Logistik (masalah klasifikasi) untuk menggabungkan prediksi sub-model dengan algoritma pembelajaran apa pun. Pendekatan ini disebut Stacking.
